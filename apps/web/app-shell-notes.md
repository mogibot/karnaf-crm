# Karnaf CRM Core - Web App Shell Notes

## Initial operator views to build first
1. Dashboard
2. Leads workspace
3. Lead detail
4. Queue center
5. System health

## Dashboard must show
- leads today
- unanswered now
- hot leads
- payment pending
- SLA risk count
- queue counts

## Lead detail must show
- lead summary
- ownership state
- status and heat
- recent transcript
- active queue items
- operator actions

## First frontend rule
Do not over-polish before data flows are stable.
The initial shell should focus on clarity and operability.
